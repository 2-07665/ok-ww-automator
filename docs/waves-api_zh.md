# Waves API

`src/ok_ww_automator/waves_api.py` 模块集成了库洛社区 API，为日常任务提供快速的、无头模式的检查。

此集成完全是可选的，并采用延迟加载的方式导入 `requests` 库。

## 功能特性

- **每日签到**: 执行社区签到操作 (`sign_in()`)。
- **实时指标**: 获取当前体力、备用体力（结晶单质）和每日活跃度积分 (`read_daily_info()`)。
- **稳健的错误处理**: 将 HTTP 和 JSON 解析错误封装成一致的结构化字典，以防止流程崩溃。

## 编排流程

如果在环境中设置了 `WAVES_API_ENABLED=true`：

1. **日常执行器 (Daily Runner)**: 
   尝试进行 API 签到并获取初始指标。这些指标将附加到运行报告中。请注意，一旦启动游戏，实时游戏读取（通过 OCR）仍会覆盖 API 获取的体力值。
   
2. **体力执行器 (Stamina Runner)**: 
   首先查询 API 以确定当前的体力水平。通过这种方式，执行器可以完全在无头模式下执行 `calculate_burn` 逻辑。**如果不需要消耗体力，automator 将完全跳过启动繁重的游戏客户端。** 如果 API 请求失败或被禁用，执行器将优雅地回退到启动游戏并使用 OCR 读取体力。
